  1. Client sends an HTTP request: A user enters a URL in their browser and the browser sends an HTTP GET request to the server.
  2. URL routing: Django checks the requested URL against its URL configuration. If it matches, Django calls the view.
  3. View execution: The view receives the request object. It decides what to return.
  4. Template loading: Django looks for the template file.
  5. Template rendering: The template engine processes the file.
  6. Server sends response: Django returns the final content. The web server sends it back to the browser.
  7. Browser renders the page: The browser receives the HTML. Parses and displays it visually
